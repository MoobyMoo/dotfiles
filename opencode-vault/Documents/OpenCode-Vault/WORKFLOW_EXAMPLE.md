# Git Workflow Example

This file demonstrates a real git workflow with branches.

## What We Did

1. Created a feature branch: `feature/example-workflow`
2. Made changes on that branch
3. Committed them with a meaningful message
4. Will merge back to master

## Key Points

- Branches keep your work isolated
- Commits create a history you can reference
- Staging area lets you organize changes logically

This is exactly how you'll use git with your OpenCode agents!
